= Module 2: Develop and build
:source-highlighter: rouge
:toc: macro
:toclevels: 1

In Module 1, you got your first AI chatbot running locally -- but you noticed the local model is small and the responses aren't great. At Parasol Insurance, the platform engineering team has set up a Model as a Service (MaaS) endpoint on OpenShift AI, serving a much more capable model that's been approved for company use. No more shadow AI -- Parasol developers can access powerful models through a governed, approved service.

In this module, you'll customize the chatbot's behavior, rebuild the container image, and connect it to Parasol's MaaS endpoint. Then you'll push your changes to the Git repository, which will trigger the automated CI/CD pipeline in Module 3.

== Learning objectives

By the end of this module, you'll be able to:

* Inspect and modify the source code of an AI Lab recipe
* Customize a chatbot's system prompt to change its behavior
* Build a new container image using Podman Desktop
* Run a containerized application locally with environment variables for MaaS connectivity
* Push application code to a Git repository to trigger CI/CD

== Exercise 3: Push your changes to Gitea

Your customized chatbot works great locally. Now it's time to push the changes to the Git repository that was scaffolded by Red Hat Developer Hub in Module 1. This push will trigger the automated CI/CD pipeline.

=== Configure the Git remote

The recipe source code was cloned locally by AI Lab. You need to add your Gitea repository (scaffolded by RHDH) as a remote so you can push your changes there.

. Open a terminal in VS Code (or use the terminal on the VM).

. Navigate to the recipe source code directory:
+
[source,bash]
----
cd ~/ai-lab-recipes/chatbot
----
+
NOTE: The exact path may vary depending on where AI Lab cloned the recipe. Check the path shown in VS Code.

. Check the current Git status:
+
[source,bash]
----
git status
----
+
You should see your modified system prompt file listed as changed.

. Add your Gitea repository as a new remote:
+
[source,bash]
----
git remote add parasol {gitea_url}/{user}/parasol-chatbot.git
----
+
NOTE: Replace the repository URL with the actual Gitea repository URL from your Red Hat Developer Hub provisioning. You can find this link in the RHDH component page.

. Stage and commit your changes:
+
[source,bash]
----
git add .
git commit -m "Customize chatbot with pirate voice personality and MaaS configuration"
----

. Push to Gitea:
+
[source,bash]
----
git push parasol main
----
+
When prompted, enter your credentials:
+
* **Username**: `{user}`
* **Password**: `{password}`

=== Verify the push

. Open your Gitea repository in a browser to confirm the code was pushed:
+
link:{gitea_url}/{user}/parasol-chatbot[Gitea Repository^]
+

* Inspected and modified the source code of an AI Lab recipe using VS Code
