= Module 1: Provisioning and exploration
:source-highlighter: rouge
:toc: macro
:toclevels: 1

It's your first day working on AI initiatives at Parasol Insurance. Normally, getting a new developer productive takes weeks -- setting up environments, requesting access, installing tools, and navigating endless documentation. But Parasol has adopted Red Hat Developer Hub to eliminate that friction.

In this module, you'll self-service a fully configured development workstation, explore the link:https://developers.redhat.com/products/red-hat-build-podman-desktop[Red Hat build of Podman Desktop^] and its extensions, and get hands-on with the AI Lab to see how developers can safely experiment with AI models and recipes on the desktop.

== Learning objectives

By the end of this module, you'll be able to:

* Use Red Hat Developer Hub to self-service a RHEL 10 virtual machine with pre-installed developer tools
* Navigate the Red Hat build of Podman Desktop and explore its extensions
* Install and use the AI Lab extension to experiment with AI models and recipes
* Understand how the AI Lab recipe catalog provides inspiration for building AI-powered applications

== Exercise 1: Provision your development environment with Red Hat Developer Hub

Your team lead has told you: "Just go to Developer Hub, pick the AI developer workstation template, and you'll have everything you need." No tickets, no waiting, no manual setup.

You'll use Red Hat Developer Hub to provision a RHEL 10 virtual machine that comes pre-configured with the Red Hat build of Podman Desktop and other essential tools.

=== Access Red Hat Developer Hub

. Click the **Developer Hub** tab at the top of the right pane.

. Click Sign in, and log in using your credentials:
+
* **Username**: `{user}`
* **Password**: `{password}`
+
image::rhdh-login.png[Red Hat Developer Hub login page,link=self,window=blank,width=700]

. After logging in, you'll land on the Red Hat Developer Hub home page. Take a moment to look around -- this is the central hub where Parasol's developers discover and provision everything they need.
+
image::rhdh-home.png[Red Hat Developer Hub home page,link=self,window=blank,width=700]

=== Create your AI developer workstation

. Click the **(+)** button at the top right of the page, then select the **AI App Dev - Parasol Chat** template from the catalog.
+
image::rhdh-template-catalog.png[Template catalog showing AI App Dev - Parasol Chat,link=self,window=blank,width=700]

. Fill in the template form with the following values:
+
* **Application Name**: Leave as the default (`ai-app-dev`)
* **Virtual Machine Name**: Leave as the default (`developer-workstation`) or enter a custom name
+
NOTE: The virtual machine name must contain only lowercase letters, numbers, and hyphens, and must start and end with an alphanumeric character.
+
image::rhdh-template-form.png[AI App Dev template form,link=self,window=blank,width=700]

. Click **Review** to review your selections, then click **Create** to begin provisioning.
+
image::rhdh-provisioning.png[Provisioning in progress,link=self,window=blank,width=700]

. Wait for the provisioning to complete. Red Hat Developer Hub is doing several things behind the scenes:
+
* Provisioning a RHEL 10 virtual machine via OpenShift Virtualization (KubeVirt)
* Installing the Red Hat build of Podman Desktop
* Scaffolding a Git repository in GitLab for your application code
* Configuring CI/CD pipelines for your project
+
TIP: This process takes a few minutes. In a traditional setup, this would take days or weeks of manual work across multiple teams.

. Once provisioning is complete, you'll see a success page with links to your new resources.
+
NOTE: The virtual machine may take a minute or two to fully start up after provisioning completes. You can continue exploring the success page and Developer Hub while it boots.
+
image::rhdh-provisioning-complete.png[Provisioning complete with resource links,link=self,window=blank,width=700]

=== Verify your provisioned resources

. From the success page, note the links to:
+
* The **GitLab Repository** where your scaffolded application code lives (we'll visit this in a bit)
* A link to **Open in Catalog**. The catalog is where all your provisioned resources are registered in Developer Hub.
+
. Click on **Open in Catalog** to view its details page.
+
image::rhdh-component-overview.png[Component overview in Red Hat Developer Hub,link=self,window=blank,width=700]

. Note the new links provided on the component page:
+
* **Virtual Machine**: A link into the OCP console to view details about your provisioned VM, including its status, resources, and connection info
* **VNC Console**: Connect to your new virtual machine's desktop environment directly from Developer Hub

=== Verify

You should now have:

* A RHEL 10 virtual machine running on the cluster
* A GitLab repository with scaffolded application code
* A component registered in Red Hat Developer Hub

== Exercise 2: Explore the Red Hat build of Podman Desktop

Now that your "workstation" is provisioned, let's explore the Red Hat build of Podman Desktop -- the primary tool Parasol developers use for container-native development on the desktop.

=== Connect to your virtual machine

NOTE: The virtual machine in this workshop simulates a real developer desktop. All the tools you'll use here -- including Podman Desktop, VS Code, and the AI Lab extension -- can be installed directly on your own laptop. Podman Desktop is available for Linux, Windows, and Mac. We're using a VM in this workshop so you don't need to install anything locally.

. In the Developer Hub component page, click the **VNC Console** link to open a VNC session to your virtual machine.
+
image::rhdh-vnc-link.png[VNC Console link in Developer Hub component page,link=self,window=blank,width=700]

. The VNC console opens, showing your RHEL 10 desktop.
+
image::vnc-console.png[VNC console connection to RHEL 10 VM,link=self,window=blank,width=700]

. Click on the user `labuser` and enter the password `labpassword`. After a few moments, you will land on the desktop. You can press `Skip` to skip the tour.
+
image::rhel-desktop.png[RHEL 10 desktop environment,link=self,window=blank,width=700]

=== Launch Podman Desktop

Click the "9-dot" menu at the bottom (if you do not see this, click the red fedora at the upper left).

. Find and launch the **Red Hat build of Podman Desktop** from the application menu or desktop shortcut.
+
image::podman-desktop-launch.png[Launching Podman Desktop from RHEL 10,link=self,window=blank,width=700]

. Once Podman Desktop opens, click `Skip` to skip configuration (it is already configured). Take a moment to explore the interface and the sections on the left, the most important of which are:
+
* **Dashboard**: Overview of your container environment
* **Containers**: Running and stopped containers
* **Images**: Available container images
* **Pods**: Grouped containers
* **Extensions**: Add-ons that provide additional functionality and integrations
* **Kubernetes**: Manage Kubernetes clusters and workloads
+
image::podman-desktop-dashboard.png[Podman Desktop dashboard overview,link=self,window=blank,width=700]

=== Explore extensions

. Click on **Extensions** in the left navigation to see what's available. The Red Hat build of Podman Desktop supports a rich extension ecosystem.
+
// TODO: Screenshot of extensions page
image::podman-desktop-extensions.png[Podman Desktop extensions page,link=self,window=blank,width=700]

. Click the `Catalog` tab to browse through the available extensions. You'll notice extensions for:
+
* Kubernetes and OpenShift integration
* Podman machine management
* Podman AI extensions
* And more
+
TIP: Extensions are how Podman Desktop stays lightweight while still supporting specialized workflows like AI development.

== Exercise 3: Install the AI Lab extension and explore recipes

At Parasol, developers need a safe, local way to experiment with AI -- no more shadow AI using random third-party services. The AI Lab extension for Podman Desktop provides exactly this: a governed, local-first approach to AI experimentation with a catalog of pre-built recipes to take inspiration from.

=== Install the AI Lab extension

. In the Catalog of extensions view, click the _Install_ button next to the **Podman AI Lab** extension.


. Click **Install** to add the AI Lab extension to your Podman Desktop (if if takes too long you may get a "Not responding" popup - just click _Wait_ and it should complete successfully). The extension will be installed, which may take a few moments as it pulls the necessary container images and sets up the environment.
+
image::ai-lab-installing.png[Installing AI Lab extension,link=self,window=blank,width=700]

. Once installed, you'll see a new **AI Lab** icon in the left navigation.
+
image::ai-lab-nav.png[AI Lab icon in Podman Desktop navigation,link=self,window=blank,width=700]

=== Explore the AI Lab

. Click on the **AI Lab** icon to open it. Take a moment to explore the different sections:
+
* **Catalog**: Pre-built AI recipes covering chatbots, code generation, summarization, and more
* **Recipes**: A collection of pre-built applications that you can run locally to see AI in action
* **Playground**: An interactive space to experiment with models directly
+
image::ai-lab-main.png[AI Lab main page,link=self,window=blank,width=700]

. GPU Support is not yet enabled on Linux, so click the `Don't display anymoe` button at the top to dismiss the GPU support message.
. Browse through the **Catalog** section. Notice the variety of recipes available -- different sizes, capabilities, and use cases. Developers can experiment with many different recipes to find what works best for their needs.
+
image::ai-lab-models.png[Available models in AI Lab,link=self,window=blank,width=700]

=== Run a recipe from the catalog

The recipe catalog is where the magic happens. Each recipe is a fully working AI application that you can run locally, inspect, and take inspiration from for your own projects.

. Navigate to the **Recipe Catalog** section in AI Lab.
+
// TODO: Screenshot of the recipe catalog
image::ai-lab-catalog.png[AI Lab recipe catalog,link=self,window=blank,width=700]

. Find the **Chatbot** recipe in the catalog. This is a pre-built application that provides a web-based chat interface backed by an AI model.
+
image::ai-lab-catalog-chatbot.png[Chatbot recipe in the AI Lab catalog,link=self,window=blank,width=700]

. Click on the **More Details** link on the recipe card to see its details, then click **Start** to instantiate it.
+
image::ai-lab-chatbot-start.png[Starting the chatbot recipe,link=self,window=blank,width=700]

. The default model `ibm-granite/granite-4.0-micro-GGUF` model will be selected. Click `Start ChatBot recipe` to confirm and start the recipe.

. Wait for the recipe to pull the required model and start the application. This may take a few minutes as it downloads the model and builds the containers.
+
image::ai-lab-chatbot-pulling.png[Chatbot recipe pulling model and starting,link=self,window=blank,width=700]

. Once the recipe is running, click the **Open Details** link to open the chatbot application in your browser, and then click the **Open ChatBot** button to launch the chat interface using the Firefox browser on your VM:
+
image::ai-lab-chatbot-running.png[Chatbot application Running details,link=self,window=blank,width=700]
image::ai-lab-chatbot-running2.png[Chatbot application running locally,link=self,window=blank,width=700]

. Try asking the chatbot a few questions:
+
[source,text,role=execute]
----
What are the benefits of using containers for application development?
----
+
[source,text,role=execute]
----
Can you explain what a large language model is in simple terms?
----
+
image::ai-lab-chatbot-running3.png[Chatting with the model,link=self,window=blank,width=700]

. Take a moment to appreciate what just happened: you downloaded an AI model, spun up a fully working chatbot application, and tested it -- all locally, without any cloud API keys or third-party services. This is the kind of safe experimentation that the AI Lab enables.

=== Inspect the recipe source code

The recipes in AI Lab aren't black boxes. You can inspect their source code, understand how they work, and take inspiration for building your own AI applications.

. Return to Podman Desktop and navigate to the running chatbot recipe in AI Lab.

. Click the **Open in VS Code** button on the recipe. This opens the locally cloned recipe source code in VS Code.
+
// TODO: Screenshot of the "Open in VS Code" button on the recipe
image::ai-lab-open-vscode.png[Open in VS Code button on the chatbot recipe,link=self,window=blank,width=700]

. In VS Code, explore the project structure. Notice how the application is put together:
+
* How it connects to the AI model endpoint
* How the chat interface handles conversation history
* How environment variables are used to configure the model connection
+
// TODO: Screenshot of VS Code with the recipe source code structure
image::vscode-recipe-structure.png[VS Code showing recipe source code structure,link=self,window=blank,width=700]

. This is exactly the kind of inspiration that Parasol's developers have been using. By exploring these recipes, the team has been learning patterns for building AI-powered applications -- and they've used those patterns to build something of their own, which you'll work with in the next module.

Confirm that:

* The AI Lab extension is installed in Podman Desktop
* You ran a recipe from the catalog and interacted with it

== Module summary

You've successfully completed the provisioning and exploration module -- and it only took a fraction of the time Parasol's old onboarding process would have required.

**What you accomplished:**

* Self-serviced a fully configured RHEL 10 development workstation through Red Hat Developer Hub -- no tickets, no waiting
* Explored the Red Hat build of Podman Desktop and its extension ecosystem
* Installed the AI Lab extension and explored different models and recipes
* Ran a chatbot recipe locally and inspected its source code in VS Code

**Key takeaways:**

* Red Hat Developer Hub eliminates the developer onboarding bottleneck with self-service templates
* Podman Desktop provides a powerful, extensible desktop container environment
* The AI Lab extension gives developers a governed, local-first approach to AI experimentation -- eliminating the need for shadow AI services
* The recipe catalog provides inspiration and working patterns that teams can use to build their own AI-powered applications
