= Module 3: Deploy and verify
:source-highlighter: rouge
:toc: macro
:toclevels: 1

In the previous modules, you provisioned a development workstation, built a customized AI chatbot, and pushed your code to Gitea. At most organizations, this is where the process stalls -- developers hand off code to operations teams, tickets get filed, and days (or weeks) pass before anything runs in production.

Not at Parasol. The platform engineering team has set up automated CI/CD pipelines using OpenShift Pipelines (Tekton) and OpenShift GitOps (ArgoCD). Your code push has already triggered the pipeline. In this module, you'll observe the automated build and deployment process, then verify your AI chatbot is running live on OpenShift.

== Learning objectives

By the end of this module, you'll be able to:

* Monitor an automated CI/CD pipeline through the Red Hat Developer Hub interface
* Understand how OpenShift Pipelines (Tekton) builds container images from source code
* Observe how OpenShift GitOps (ArgoCD) automatically syncs and deploys applications
* Verify a live application running on OpenShift and access it through a route

== Exercise 1: Watch the CI/CD pipeline in action

The moment you pushed your code to Gitea in Module 2, a webhook triggered an OpenShift Pipeline. This pipeline pulls your source code, builds a container image, and pushes it to the cluster's container registry. You can watch all of this from the Red Hat Developer Hub interface.

=== Understand the pipeline stages

The pipeline performs several stages automatically:

. **Fetch source**: Pulls your code from the Gitea repository
. **Build image**: Uses the `Containerfile` to build your container image (the same one you built locally in Module 2)
. **Push image**: Pushes the built image to the cluster's internal container registry
. **Update deployment manifest**: Updates the deployment configuration with the new image reference
+
// TODO: Screenshot of pipeline stages/tasks completing
image::rhdh-pipeline-stages.png[Pipeline stages completing successfully,link=self,window=blank,width=700]

. Wait for the pipeline to complete. All stages should show a green checkmark.
+
TIP: This automated pipeline replaces the 3-5 day manual handoff process at Parasol. Code is built and validated automatically, with the same Containerfile used locally and in production -- eliminating the "works on my machine" problem.

=== Verify

Confirm that:

* The pipeline run is visible in the RHDH CI tab
* All pipeline stages completed successfully (green checkmarks)
* The container image was built and pushed to the registry

== Exercise 2: Observe the GitOps deployment

With the container image built and pushed, ArgoCD takes over. OpenShift GitOps (ArgoCD) watches for changes to the deployment manifests and automatically syncs them to the cluster. This means your application gets deployed without any manual intervention.

=== View the ArgoCD sync in RHDH

. In your component view in Red Hat Developer Hub, click on the **CD** tab.

== Module summary

You've just witnessed the full power of Red Hat's desktop-to-production workflow. Code pushed from your developer workstation was automatically built, deployed, and made accessible -- without a single manual handoff.

**What you accomplished:**

* Monitored an automated CI/CD pipeline build through the RHDH CI tab
* Observed ArgoCD automatically detect and deploy the new container image through the RHDH CD tab
* Verified the live application running on OpenShift through the Topology view
* Accessed and tested the production chatbot via its public route

**Key takeaways:**

* OpenShift Pipelines (Tekton) automates the build process -- the same Containerfile used locally builds the production image
* OpenShift GitOps (ArgoCD) automates deployment -- no manual intervention between code push and production
* Red Hat Developer Hub provides a single pane of glass for CI/CD visibility
* The entire process -- from code push to production -- takes minutes, not days
* Platform engineers set up the guardrails once, and developers benefit automatically
