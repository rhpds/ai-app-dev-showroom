= AI App Development from Desktop to Production

Welcome to the AI App Development from Desktop to Production workshop!

Many generative AI projects are abandoned after proof of concept -- costing organizations billions. This is often due to a lack of foresight as well as the challenges organizations face when moving concepts to production -- mismatched dependencies, untrusted or malicious packages, unvetted Model Context Protocol (MCP) tools or large language models (LLMs), and more.

In this workshop, you will learn how Red Hat addresses these challenges with innovative new developer tools for local experimentation, and how platform engineers enforce policy and IT controls as those ideas move into production.

== What you'll learn

In this workshop, you will:

* Scaffold new AI experimentation projects using Red Hat Developer Hub
* Use Red Hat Desktop and the Red Hat build of Podman Desktop to experiment with AI apps on the desktop
* Develop and customize AI-powered applications locally with container-native workflows
* Move applications from desktop to production using OpenShift Pipelines and ArgoCD GitOps principles

== Who this is for

This workshop is designed for developers and platform engineers who want to understand how Red Hat eliminates the "production gap" that causes so many AI projects to fail.

**Experience level**: Intermediate

== Prerequisites

Before starting this workshop, you should have:

* Basic familiarity with containers and container images
* Some experience with Git and version control
* A general understanding of CI/CD concepts

No prior experience with OpenShift, Podman Desktop, or AI/ML is required.
