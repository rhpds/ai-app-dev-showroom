== Module overview

=== Module 1: Provisioning and exploration

You'll use Red Hat Developer Hub to self-service a fully configured RHEL 10 virtual machine with the Red Hat build of Podman Desktop pre-installed. Then you'll explore Podman Desktop and its extensions -- including the AI Lab extension -- and instantiate an AI chatbot recipe to chat with a local model.

**Key technologies**: Red Hat Developer Hub, RHEL 10, Podman Desktop, AI Lab extension

=== Module 2: Develop and build

After experimenting with a local model, you'll notice the performance is limited. You'll modify the chatbot's source code to customize the system prompt, rebuild the container image with Podman Desktop, and run it locally using environment variables that connect to Parasol's approved Model as a Service (MaaS) endpoint on OpenShift AI. Finally, you'll push your changes to Gitea.

**Key technologies**: Podman Desktop, VS Code, Containerfile, Gitea, OpenShift AI (MaaS)

=== Module 3: Deploy and verify

Your code push triggers an automated CI/CD pipeline. You'll watch the OpenShift Pipeline (Tekton) build your application image via the Red Hat Developer Hub CI tab, then observe ArgoCD automatically sync and deploy the application to OpenShift via the CD tab. Finally, you'll verify your AI chatbot is running live on the cluster.

**Key technologies**: OpenShift Pipelines (Tekton), OpenShift GitOps (ArgoCD), Red Hat Developer Hub
